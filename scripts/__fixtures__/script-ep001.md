## Хук (0–15 сек)
[ГОЛОС] Дизайн-система, которая якобы «росла внутри Meta восемь лет» и «питает
тринадцать тысяч приложений». Звучит как маркетинговый слоган на миллион. Но
первое, что она делает при обычной установке — тихо дёргает postinstall-скрипт.
Сегодня разбираемся, где тут заслуженная годнота, а где — красивая обёртка.
[СКРИНКАСТ: крупным планом — GitHub-страница astryx с заголовком про «8 лет в
Meta» и «13 000+ приложений», затем резкий переход на терминал с командой pnpm
install]
[МЕМ: реакция «подозрительно, но окей» — прищуренный взгляд на монитор]

## Что это за репо (15–60 сек)
[ГОЛОС] Astryx — open-source дизайн-система на React и StyleX. Обещает больше
ста пятидесяти доступных компонентов, брендовую тему, тёмный режим, готовые к
продакшну шаблоны страниц и свой CLI. Отдельная фишка — «no build plugin, no
styling library to adopt»: подключил готовый CSS и типизированные компоненты —
и всё работает. Плюс громкий тезис «одна система для людей и агентов»: и
разработчик, и AI-ассистент якобы строят интерфейсы одинаково, через один и
тот же CLI и одну документацию.
[АНИМАЦИЯ: StarChart — рост звёзд]
[СКРИНКАСТ: полистать README репозитория — список из 150+ компонентов, семь
готовых тем, раздел про CLI]

## Живой тест (основа)
[ГОЛОС] Первым делом — красный флаг: в apps/sandbox/package.json нашёлся
postinstall, который на обычной установке запустил бы `astryx init --features
agents` без спроса. Поэтому весь тест идёт с `--ignore-scripts` — и это уже
тема для отдельного разговора.
[СКРИНКАСТ: строка "postinstall": "astryx init --features agents" в
apps/sandbox/package.json крупным планом, рядом — фрагмент кода init.mjs,
который просто пишет локальные doc-файлы через fs, без сети и без eval]
[МЕМ: «сапёр» — осторожно тыкает в постинсталл-скрипт]

[ГОЛОС] Дальше — чистая установка: pnpm install с отключёнными скриптами,
двадцать шесть workspace-проектов, девятьсот четырнадцать пакетов, около
двенадцати секунд, ноль ошибок, lockfile проходит supply-chain-политику pnpm.
[СКРИНКАСТ: терминал — pnpm install --ignore-scripts выполняется чисто за
~12 секунд, в конце — строка о прохождении supply-chain-политики]

[ГОЛОС] CLI реально запускается без сборки, как и обещано. Смотрим документацию
компонента — не заглушка, а честная карточка: анатомия, best practices,
таблица пропсов с типами и дефолтами. И это не единичный пример — в
packages/core/src физически лежит сто семнадцать директорий компонентов.
[СКРИНКАСТ: запуск node packages/cli/bin/astryx.mjs component Button --dense,
полный вывод карточки компонента на экране]

[ГОЛОС] Проверяем самодиагностику — astryx doctor: четыре проверки, ноль
провалов, версии core и cli синхронны, зависимости на месте.
[СКРИНКАСТ: вывод astryx doctor — "4 passed, 0 failures"]

[ГОЛОС] Публикуемые пакеты собираются начисто: build-пакет и core-пакет,
StyleX компилирует тысячи правил в один CSS-файл плюс UMD-бандл. Но первая
попытка собрать build напрямую командой node build.mjs — упала с "tsc: command
not found". Не баг репозитория, а моя ошибка запуска: без pnpm PATH не видит
node_modules/.bin. Через pnpm -F build build всё собирается нормально.
[СКРИНКАСТ: сначала падение "tsc: command not found" при прямом node build.mjs,
затем успешная сборка через pnpm -F @astryxdesign/build build]

[ГОЛОС] И кульминация теста — тесты. Канонический pnpm test из корня
репозитория: триста двадцать девять файлов, шесть тысяч сто сорок тестов —
и все прошли. Ноль провалов. Это очень крупный и живой набор для дизайн-системы.

## Где README врёт
[ГОЛОС] По инженерной части README почти не врёт — все ключевые обещания
подтвердились в песочнице. Но есть два места, где стоит присмотреться. Первое —
technical: 28 из 6140 тестов падали при запуске через pnpm -F core test с
ошибкой ENOENT на packages/core/src — не потому что тесты сломаны, а потому что
CLI-тесты читают путь относительно cwd, а vitest root смотрит на две папки
выше. Задокументированная команда pnpm test из корня — единственный честный
способ всё перепроверить, и README это нигде явно не подчёркивает.
[СКРИНКАСТ: сравнение двух команд рядом — pnpm -F @astryxdesign/core test с
28 падениями и pnpm test из корня с полностью зелёным результатом]

[ГОЛОС] Второе — маркетинговое: «восемь лет внутри Meta» и «тринадцать тысяч
плюс приложений» — это цифры, которые невозможно проверить из кода. Само по
себе это не обман, но именно такие цифры обычно и создают хайп вокруг
репозитория, а не реальная инженерная работа, которая здесь, объективно,
сделана добротно.
[СКРИНКАСТ: снова заголовок README с цифрами «8 лет» и «13 000+ приложений» —
на этот раз с более скептической интонацией голоса]
[SHORT cut: разоблачение непроверяемых маркетинговых цифр — «8 лет в Meta» и
«13 000+ приложений» — против того, что реально можно пощупать в коде]

## Вердикт
[ГОЛОС] Итог: Astryx делает то, что обещает, и делает это на уровне зрелого
продукта, а не демо. Чистая установка, CLI без сборки, сто семнадцать
компонентов с реальной документацией, публикуемые пакеты собираются начисто, и
шесть тысяч сто сорок тестов проходят без единого провала. Настоящих поломок
продукта не нашлось — только мои ошибки запуска. Флаг postinstall при разборе
исходника оказался безобидным. Единственная оговорка — непроверяемые из кода
маркетинговые цифры, которые тянут часть истории в сторону хайпа. Но по сути
это крепкая, живая, честно работающая дизайн-система.
[СКРИНКАСТ: экран с итоговым списком — установка, CLI, доки, сборка, тесты — все
пункты отмечены галочкой]
