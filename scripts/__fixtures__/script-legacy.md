## Хук (0–15 сек)
[ГОЛОС] Библиотека компонентов, которая якобы «выросла из внутреннего фреймворка
крупного продукта» и «крутится в тысячах приложений». Звучит как маркетинговый
слоган на миллион. Но первое, что она делает при обычной установке — тихо дёргает
postinstall-скрипт. Сегодня разбираемся, где тут заслуженная годнота, а где —
красивая обёртка.
[СКРИНКАСТ: крупным планом — GitHub-страница репозитория с заголовком про
«годы внутри крупной команды» и «тысячи приложений», затем резкий переход на
терминал с командой pnpm install]
[МЕМ: реакция «подозрительно, но окей» — прищуренный взгляд на монитор]

## Что это за репо (15–60 сек)
[ГОЛОС] WidgetKit — open-source библиотека UI-компонентов на React. Обещает
больше сотни доступных компонентов, брендовую тему, тёмный режим, готовые к
продакшну шаблоны страниц и свой CLI. Отдельная фишка — «no build step»:
подключил готовый CSS и типизированные компоненты — и всё работает.
[АНИМАЦИЯ: StarChart — рост звёзд]
[СКРИНКАСТ: полистать README репозитория — список компонентов, готовые темы,
раздел про CLI]

## Живой тест (основа)
[ГОЛОС] Первым делом — красный флаг: в одном из workspace-пакетов нашёлся
postinstall, который на обычной установке запустил бы генератор без спроса.
Поэтому весь тест идёт с `--ignore-scripts` — и это уже тема для отдельного
разговора.
[СКРИНКАСТ: строка postinstall в package.json крупным планом, рядом — фрагмент
кода генератора, который просто пишет локальные doc-файлы через fs, без сети и
без eval]
[МЕМ: «сапёр» — осторожно тыкает в постинсталл-скрипт]

[ГОЛОС] Дальше — чистая установка: pnpm install с отключёнными скриптами,
проходит быстро и без ошибок, lockfile проходит supply-chain-политику pnpm.
[СКРИНКАСТ: терминал — pnpm install --ignore-scripts выполняется чисто, в конце —
строка о прохождении supply-chain-политики]

[ГОЛОС] CLI реально запускается без сборки, как и обещано. Смотрим документацию
компонента — не заглушка, а честная карточка: анатомия, best practices, таблица
пропсов с типами и дефолтами.
[СКРИНКАСТ: запуск CLI-команды генерации компонента, полный вывод карточки
компонента на экране]

[ГОЛОС] И кульминация теста — тесты. Канонический pnpm test из корня репозитория:
сотни файлов, тысячи тестов — и все прошли. Ноль провалов.

## Где README врёт
[ГОЛОС] По инженерной части README почти не врёт — все ключевые обещания
подтвердились в песочнице. Но есть два места, где стоит присмотреться. Первое —
technical: часть тестов падала при запуске отдельного пакета не из корня, с
ошибкой ENOENT — не потому что тесты сломаны, а потому что CLI-тесты читают путь
относительно cwd, а vitest root смотрит на две папки выше. Задокументированная
команда pnpm test из корня — единственный честный способ всё перепроверить, и
README это нигде явно не подчёркивает.
[СКРИНКАСТ: сравнение двух команд рядом — запуск отдельного пакета с падениями и
pnpm test из корня с полностью зелёным результатом]

[ГОЛОС] Второе — маркетинговое: «годы внутри крупной команды» и «тысячи
приложений» — это цифры, которые невозможно проверить из кода. Само по себе это
не обман, но именно такие цифры обычно и создают хайп вокруг репозитория, а не
реальная инженерная работа, которая здесь, объективно, сделана добротно.
[СКРИНКАСТ: снова заголовок README с непроверяемыми цифрами — на этот раз с более
скептической интонацией голоса]
[SHORT cut: разоблачение непроверяемых маркетинговых цифр против того, что реально
можно пощупать в коде]

## Вердикт
[ГОЛОС] Итог: WidgetKit делает то, что обещает, и делает это на уровне зрелого
продукта, а не демо. Чистая установка, CLI без сборки, компоненты с реальной
документацией, публикуемые пакеты собираются начисто, и тысячи тестов проходят
без единого провала. Настоящих поломок продукта не нашлось — только мои ошибки
запуска. Флаг postinstall при разборе исходника оказался безобидным. Единственная
оговорка — непроверяемые из кода маркетинговые цифры, которые тянут часть истории
в сторону хайпа. Но по сути это крепкая, живая, честно работающая библиотека.
[СКРИНКАСТ: экран с итоговым списком — установка, CLI, доки, сборка, тесты — все
пункты отмечены галочкой]
