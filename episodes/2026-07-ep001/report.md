# Review: astryx

**URL:** https://github.com/facebook/astryx
**Reviewed:** 2026-07-11

## Заявлено (Claimed)
Astryx позиционируется как open-source дизайн-система «выросшая внутри Meta за 8 лет»,
якобы самая используемая и крупная дизайн-система компании, «питающая 13 000+ приложений».
Обещания из README: 150+ доступных (accessible) React-компонентов, brand-level темизация,
тёмная тема, готовые к отгрузке шаблоны страниц и CLI — всё как единая система.
Ключевые тезисы: собрано на React + StyleX; потребитель просто импортирует готовый CSS
и типизированные компоненты — «no build plugin, no styling library to adopt»; открытые
внутренности (компоненты композируются на любом уровне, `swizzle` извлекает исходник
компонента в проект); отсутствие styling lock-in (переопределение через `className` —
Tailwind / CSS-модули / обычный CSS); тема — это набор переопределений CSS-переменных;
и «одна система для людей и агентов» — API, доки и CLI спроектированы так, чтобы человек
и AI-ассистент строили одинаково. Заявлено 7 готовых тем и CLI с командами для доков
компонентов, шаблонов, тем и codemod-миграций.

## Работает (Works)
Проверено в песочнице `episodes/2026-07-ep001/sandbox/astryx` (Node 22.22, pnpm 11.10),
установка `pnpm install --ignore-scripts` (скрипты отключены осознанно — см. «Подозрительно»):

- **Установка**: `pnpm install --ignore-scripts` прошла чисто за ~12 с, 26 workspace-проектов,
  914 пакетов, lockfile прошёл supply-chain-политику pnpm. Ноль ошибок.
- **CLI работает без сборки** (как и заявлено в CLAUDE.md): `node packages/cli/bin/astryx.mjs help`
  выдаёт полный список команд (component, docs, swizzle, template, layout, upgrade, theme,
  search, build, doctor и т.д.).
- **Доки компонентов реальны и подробны**: `astryx component --list` показывает большую
  сгруппированную библиотеку; `astryx component Button --dense` выдаёт полноценную карточку —
  анатомия, best practices (Do/Don't), полная таблица props с типами и дефолтами.
  В `packages/core/src` физически лежит 117 директорий-компонентов.
- **`astryx doctor`**: 4 проверки пройдено, 0 провалов (верно определил Node, версии
  core↔cli в синхроне v0.1.4, peer-зависимости StyleX/React удовлетворены).
- **`astryx search modal`** возвращает осмысленный ранжированный список (AlertDialog,
  CommandPalette, Dialog...). **`astryx template --list`** выдаёт богатый набор готовых
  шаблонов (AI Chat, Analytics Dashboard, Login, Checkout и др.).
- **Сборка публикуемых пакетов проходит** (через pnpm, чтобы `node_modules/.bin` был в PATH):
  `@astryxdesign/build` собрался (`dist/vite.mjs` + `.d.ts`); `@astryxdesign/core` собрался
  начисто — Babel скомпилировал 477 файлов, StyleX собрал 5445 правил в `astryx.css` (118 КБ),
  плюс UMD-бандл `dist/astryx.umd.js`.
- **Тесты**: канонический `pnpm test` из корня репозитория — **329 файлов, 6140 тестов,
  все прошли (0 провалов)** за ~70 с. Это очень крупный и живой набор.

## Сломалось (Broke)
Настоящих поломок продукта не найдено. Все «сбои» оказались артефактами моего способа
запуска, а не багами репозитория:

- Первая попытка собрать `@astryxdesign/build` командой `node build.mjs` напрямую упала с
  `/bin/sh: tsc: command not found` (`Error: Command failed: tsc --project tsconfig.build.json`,
  status 127). Причина — прямой запуск не кладёт `node_modules/.bin` в PATH; через
  `pnpm -F @astryxdesign/build build` всё собирается.
- Первый прогон тестов через `pnpm -F @astryxdesign/core test` дал 28 падений из 6140, все
  с одинаковой ошибкой `ENOENT: no such file or directory, scandir 'packages/core/src'`
  (в `packages/cli/src/lib/component-discovery.mjs`). Причина — pnpm ставит cwd в
  `packages/core`, а vitest root указывает на `../..`, и эти CLI-тесты читают путь
  `packages/core/src` относительно cwd. При запуске документированной командой `pnpm test`
  из корня — все 6140 зелёные.
- Мелкие некритичные предупреждения в логах тестов: React `act(...)` warning в тестах
  Table/columnResize и «Promise ... was not awaited» в `scripts/build-css.test.mjs` (тест
  всё равно проходит, но сломается на Vitest 3). На вердикт не влияют.

## Подозрительно (Suspicious)
- **Флаг safety-gate (обработан)**: в `apps/sandbox/package.json:14` есть
  `"postinstall": "astryx init --features agents"`. Это lifecycle-скрипт, который на обычном
  `pnpm install` выполнился бы автоматически. **Поэтому весь ревью проведён с
  `--ignore-scripts`** — флагнутый `postinstall` (и husky-хук `"prepare": "husky install"`
  в корневом `package.json:34`) НЕ запускались, по явному одобрению хоста. Разбор исходника
  `packages/cli/src/commands/init.mjs` показал, что с `--features agents` команда лишь пишет
  локальные doc-файлы (AGENTS.md/CLAUDE.md) через `fs` — без сети, без eval/base64, без
  скачивания бинарников. То есть флаг сработал по форме (postinstall), но полезная нагрузка
  выглядит доброкачественной.
- Ни `curl … | sh` / `wget … | bash`, ни обфусцированных/base64/eval-блобов, ни странных
  бинарников в репозитории не обнаружено. `https://`-ссылки в `.github/workflows/*` — это
  штатный CI (push на gh-pages по токену, publish в npm-registry), выполняется в GitHub
  Actions, а не в нашей локальной установке.
- **Star-forensics**: полноценно оценить динамику звёзд из shallow-клона нельзя (нет истории/
  метаданных GitHub). Но косвенные признаки говорят ПРОТИВ накрутки: номера PR доходят до
  #3819, 67 незакрытых changeset-файлов, 14 CI-workflow, последний коммит — сегодняшним
  числом, 117 директорий компонентов в core, 6140 проходящих тестов. Такой объём и живость
  несовместимы с пустой звёздной накруткой. Единственное, что стоит проверить редакционно, —
  громкость маркетинга («8 лет внутри Meta», «13 000+ приложений»): это непроверяемые из
  кода утверждения, и именно они — потенциальный источник хайпа, а не сам код.

## Черновой вердикт (Draft verdict lean)
**ГОДНОТА** — Astryx делает то, что обещает, и делает это на уровне зрелого продукта, а не
демо. Установка чистая, CLI запускается без сборки и выдаёт реальные, подробные доки на 117+
компонентов, публикуемые пакеты (`build`, `core`) собираются начисто (StyleX → 118 КБ CSS +
UMD-бандл), а канонический `pnpm test` проходит все 6140 тестов в 329 файлах без единого
провала. Ни одного настоящего дефекта продукта в песочнице не всплыло — все «поломки» были
моими ошибками вызова (PATH без `.bin`, cwd-чувствительные CLI-тесты). Флаг safety-gate
(`postinstall: astryx init`) при разборе исходника оказался безобидным писателем локальных
doc-файлов. Единственная оговорка, удерживающая от безоговорочной оценки, — непроверяемый из
кода маркетинг («8 лет в Meta», «13 000+ приложений»), который тянет часть истории в сторону
ХАЙПа; но по инженерной сути — это крепкая, живая, честно работающая дизайн-система. Черновой
вывод: ГОДНОТА с редакционной задачей отдельно проверить громкие маркетинговые цифры.
